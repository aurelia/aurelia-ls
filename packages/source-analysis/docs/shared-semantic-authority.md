# Shared Semantic Authority

Use this note when adding a consumer, claim family, cache, projection, invalidation rule, or public contract.

Paired with [governing-axes.md](./governing-axes.md):

- `governing-axes` keeps capability classification honest
- this note keeps the product center honest

## Core Burden

Make useful semantic judgments about Aurelia programs whose full behavior is not uniformly statically knowable, without bluffing what is unknown, inapplicable, stale, withdrawn, or too costly for a given consumer to spend.

## Authority Record

Write these down before committing to the surface:

| Question | What must stay separate |
| --- | --- |
| What knowledge is preserved here? | preserved knowledge |
| What can be honestly claimed now? | admissible claim |
| What may this consumer safely act on? | consumer spend |
| What is the honest non-positive outcome? | no-claim, refusal, silence, blocked negative, withdrawal |
| What else must converge for truth? | declarations, manifests, registrations, product-authored knowledge |
| What would force prior closure to reopen? | retreat under change |
| Can this be supported under live conditions? | sustainable closure, not demo closure |

## Product Rules

1. Many consumers should consult one semantic authority instead of rebuilding meaning independently.
2. Preserved knowledge, admissible claim, and consumer action are different thresholds.
3. No-claim outcomes are first-class product behavior.
4. Presence, reachability, and meaning are different burdens.
5. Declarations belong to the same truth system as analysis.
6. Truth must retreat under change.
7. A capability may be promised only if it can be sustained honestly under live, incremental, resource-bounded conditions.

## Operational Use

When a change is non-trivial, force it through this sequence:

1. Name the shared authority that should own the truth.
2. State what it preserves even when it cannot yet claim it.
3. State what it can honestly claim now.
4. State what this consumer may safely spend.
5. State the honest no-claim outcomes.
6. State what declarations must converge with analysis.
7. State what changes force prior closure to retreat.
8. State the live support ceiling.

If the consumer needs to re-derive meaning that another surface already derived, stop and pull that truth back toward shared authority.

## Drift Signatures

These usually mean the product root is drifting:

- Local truth-making: consumers or hosts rebuild meaning instead of consulting shared authority.
- Stage resurrection: the system starts behaving as if one produced semantic object is handed from phase to phase.
- Universal readiness: different consumers are treated as if they share one consequence threshold.
- Positive-answer bias: every position is pushed toward a weaker positive instead of sometimes returning no claim.
- Frozen closure: earlier success is treated as if withdrawal and reopening are secondary.
- Flat platform drift: inspection, adjudication, authoring, emission, reporting, and runtime projection get flattened into one vague family.
- Demo-driven ambition: capability naming is based on what can be shown once, not what can be sustained live.

## Questions For Autonomous Work

- Is this surface consulting shared authority or rebuilding truth locally?
- What is preserved here, what is claimable here, and what is spendable by this consumer?
- Is the honest outcome here a claim, a refusal, a silence, or a no-claim?
- Are presence, reachability, and meaning being collapsed into one lookup?
- What declarations have to converge with analysis for this answer to be truthful?
- What changes would force prior closure to retreat?
- Is this capability supportable under live product conditions?

## Quick Examples

- A snapshot can preserve useful knowledge without being the product object that owns truth.
- A route witness may be claimable for audit while still being too weak for automated transformation.
- A stale or invalidated semantic answer should withdraw or degrade, not silently persist.
- A manifest or registration fact that changes the answer belongs inside the authority system, not in an external exception pile.
