# Open Tensions

Some tensions are not resolved yet and should stay visible:

- broad cheap structural recovery versus deeper semantic understanding
- stable public contracts versus a fast-changing internal model
- framework-agnostic substrate versus future Aurelia-specific meaning
- JSON snapshots as a clean contract versus richer live state for fast consecutive queries
- human steering versus hidden heuristics

There is also a more immediate practical tension: the package already has tech debt. Some of the old reader surfaces still shape the code too much, some newer features are not yet spent all the way through, and some layers are cleaner in theory than in implementation.

Those are not reasons to stop. They are part of the work. The package should get better at exposing these tensions in a usable form, not just carrying them silently.
